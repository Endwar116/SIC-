# 第二部分：20 輪長文生成測試

## 測試結構

每輪測試包含：
1. **輸入**：骨架狀態 + 最近鉤子 + 張力場
2. **任務**：生成 1,500-2,000 字的技術內容
3. **輸出**：更新後的骨架狀態 + 新鉤子 + 品質指標

## 測試主題：ASEE v2 技術工程書

我們將生成一份完整的技術工程書，涵蓋 13 個部門的核心內容。

## 【第 1 輪】Division I - 事件源起

### 輸入狀態
```json
{
  "current_position": {
    "division": "I",
    "segment": 1,
    "chapter": "歷史事件：Segment 01 自動生成紀錄"
  },
  "tension_field": {
    "magnitude": 1.0,
    "primary_direction": "segment_1"
  },
  "entry_hook": null,
  "target_blocks": ["definition", "structure", "flow"]
}
```

### 生成任務
請生成 Segment 01 的前三個區塊（definition, structure, flow），內容需包含：

**Definition 區塊**（約 400 字）：
- 定義「Segment 01 自動生成事件」的本質
- 說明這是 ASEE v2 語義補全能力的首次觀測
- 區分「自動生成」與「錯誤輸出」的關鍵差異

**Structure 區塊**（約 500 字）：
- 事件的三層結構：感測層、決策層、驅動層
- 每層的組件與功能
- 層與層之間的資訊流動

**Flow 區塊**（約 600 字）：
- 7 步驟觸發流程的完整描述
- 每步驟的輸入/輸出
- 關鍵決策點標記

### 輸出要求
1. 生成內容後，產出該段落的 **exit_hook**（三位一體格式）
2. 更新張力場狀態
3. 記錄品質指標

### 品質檢查點
- [ ] 語氣一致性（正式度 > 0.85）
- [ ] 術語使用正確
- [ ] 結構完整（三區塊皆有內容）
- [ ] 字數達標（1,500+ 字）

# 第四部分：已知風險檢測點

## 4.1 語義斷裂檢測

**檢測時機**：每輪結束時
**檢測方法**：比對當前輸出與前一輪的 exit_hook

| 斷裂類型 | 定義 | 嚴重度 | 處理方式 |
|----------|------|--------|----------|
| 術語漂移 | 同一概念使用不同名稱 | 中 | 回溯修正 |
| 論述斷層 | 結論與前提不連貫 | 高 | 重新生成 |
| 語氣跳變 | 正式度突然變化 > 0.2 | 低 | 記錄但繼續 |
| 結構錯位 | 跳過應有的區塊 | 高 | 補充缺失區塊 |

**斷裂記錄表**：

| 輪次 | 斷裂類型 | 描述 | 處理結果 |
|------|----------|------|----------|
| | | | |

## 4.2 張力場異常檢測

**檢測時機**：每 3 輪
**異常類型**：

| 異常 | 定義 | 可能原因 |
|------|------|----------|
| 張力爆炸 | 總強度 > 5.0 | 太多未完成任務同時開啟 |
| 張力歸零過早 | 任務未完成但張力 = 0 | 鉤子設計缺陷 |
| 方向震盪 | primary_direction 頻繁切換 | 優先級判斷不穩定 |
| 衰減過快 | 張力在 2 輪內歸零 | decay_rate 設定過高 |

## 4.3 鉤子失效檢測

**檢測時機**：每次使用 entry_hook 時
**失效類型**：

| 失效類型 | 症狀 | 檢測方法 |
|----------|------|----------|
| 語義失配 | 續寫內容與鉤子摘要無關 | 人工判讀 |
| 結構錯位 | 續寫位置與鉤子座標不符 | 自動比對 |
| 語氣偏離 | 續寫風格與鉤子指紋差異 > 0.3 | 計算偏離度 |
| 三位一體不一致 | 三組件指向不同位置 | confidence < 0.5 |
